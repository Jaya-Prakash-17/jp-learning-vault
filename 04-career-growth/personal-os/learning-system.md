# The Lightweight Learning System

As a Senior Integration Engineer, you are bombarded with new technologies daily. You cannot learn everything. Use this system to filter the noise.

## Core Philosophy
- **Just-In-Time Learning**: Learn the exact AWS command you need to fix the current ticket today.
- **Just-In-Case Learning**: Watching 40 hours of Kubernetes tutorials when your company only uses on-prem VMs. **Avoid this.**

## The Workflow

### 1. Capture (The Scratchpad)
When you find a useful command or fix on StackOverflow/Documentation, drop it into a temporary text file or sticky note. Do not format it yet.

### 2. Condense (The Vault)
During your Friday Weekly Review, move the note into the `jp-learning-vault`. 
- **Rule 1**: Never copy-paste blindly. Rewrite the solution in your own words. If you can't explain it simply, you don't understand it.
- **Rule 2**: Put it in the right place. Is it a command? Put it in `99-cheatsheets/`. Is it a broad concept? Put it in `02-core-knowledge/`. 

### 3. Discard
If the information is highly specific to a legacy system you will never touch again (e.g., a weird quirk in Edifecs v8.4 that is patched in v9.0), do not put it in the vault. The vault is for career-spanning knowledge, not ephemeral trivia.
