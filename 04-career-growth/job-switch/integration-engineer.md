# Role Profile: Healthcare Integration Engineer

## The Role
While a Consultant focuses on the business rules and visual mapping, an Integration Engineer focuses on the raw code, APIs, and infrastructure that move the data. You are less concerned with "What does this X12 segment mean?" and more concerned with "How do I process 10,000 of these per second without crashing the JVM?"

## Core Responsibilities
- Writing complex Groovy/Java scripts to orchestrate API calls.
- Designing synchronous REST/SOAP pipelines.
- Tuning middleware servers (thread pools, memory allocation).
- Managing deployments via Git and CI/CD pipelines.
- Troubleshooting Linux servers and Docker containers.

## Why You Want This
- **Technical Depth**: This role pulls you out of proprietary, niche tools (like MapBuilder) and forces you to use widely applicable software engineering tools (Java, Docker, Git, Linux). 
- **Future Proofing**: If Edifecs were to disappear tomorrow, a Consultant might struggle to pivot. An Integration Engineer can immediately take a job at any tech company because their skills (API orchestration, JVM tuning, Linux) are universal.

## The Interview Focus
To pass an interview for this role, you must demonstrate:
1. **Coding Ability**: You must be able to write actual code (Groovy/Python/Java) to parse JSON/XML and handle HTTP exceptions.
2. **Infrastructure Knowledge**: Expect heavy questions on Linux CLI (`grep`, `tail`, `netstat`) and containerization (Docker).
3. **Performance Tuning**: They will ask how you handle memory leaks, thread lockups, and high-latency bottlenecks. You must use your Humana 80% TAT reduction as your primary weapon here.

## Target Keywords for Resume
- Groovy / Java / Python
- RESTful APIs / SOAP
- JSON / XML / XML DOM parsing
- Linux CLI
- Git / CI/CD
- High-Throughput Orchestration
